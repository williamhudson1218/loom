# Loom

A local dashboard for managing your [Claude Code](https://claude.com/claude-code) sessions — a live, color-coded triage board that indexes every chat, summarizes it with AI, tells you which ones are running and whose turn it is, and lets you jump to, resume, close, and even message any session straight from the UI. Ships as a macOS menu-bar app.

> Built for a single-developer workflow with many concurrent Claude Code sessions across tmux panes. macOS + tmux focused.

## Why

If you run a lot of Claude Code sessions, the built-in resume list is too shallow to find anything and the auto-titles go stale. Loom indexes **all** your transcripts (`~/.claude/projects/`), generates a proper **title / overview / key-moments / state** for each, and surfaces them as a board you can actually triage.

## Features

- **AI summaries** — each session gets a specific title, one-line overview, key-moments bullets, and a state (`done` / `your turn` / `caveats` / `error`), generated by a headless `claude -p` pass.
- **Live detection** — knows which sessions are *actually running* right now (process-tree based) and in which tmux pane, with a ⚡ *working* indicator (actively generating) vs *your turn* (idle, waiting on you).
- **Jump / resume / branch / close** — open a live chat's tmux pane in Ghostty, resume a stale one into an empty pane (picks the pane by session·window·position), **branch** any chat (`claude --resume … --fork-session`) into an empty pane to fork its context into a new session while leaving the original untouched, or close one to free its pane.
- **Built-in chat panel** — click a card to read the transcript and send messages straight into the running session.
- **Crash recovery** — captures a full tmux layout snapshot every 15s; `restore` rebuilds every session with exact pane geometry and re-resumes the right chats after a tmux crash.
- **PR links, activity heatmap, triage filters, auto-refresh** — and a menu-bar app with live counts.

## Architecture

```
src/            shared TypeScript backend
  parser        JSONL transcript -> fields (incl. PR detection, activity)
  indexer       mtime-gated index of ~/.claude/projects/*, 7-day window
  analyzer      headless `claude -p --safe-mode` -> {title, overview, state, key_moments}
  placements    live-session detection (placement hook + tmux process tree)
  goto          jump / resume / close / send-keys into panes
  snapshot      full tmux-layout capture for crash recovery
  restore       rebuild sessions + geometry after a crash
  server        localhost HTTP server: dashboard + JSON API (/api/data, /goto, /resume, /close, /send, ...)
  dashboard     self-contained HTML/JS dashboard (served by the server)
app/            Electron app — bundles the backend, embeds the server, tray + window, daemon intervals
hooks/          SessionStart/UserPromptSubmit hook that records each chat's tmux pane
```

The backend runs as a daemon (the app), serves the dashboard at `localhost:4317`, and does all privileged work (tmux, ps, osascript, file reads, spawning `claude -p`).

## Requirements

- macOS, [tmux](https://github.com/tmux/tmux), and Claude Code (the `claude` CLI, logged in)
- Node 20+ (the CLI uses a native `better-sqlite3`; the app rebuilds it for Electron)
- Optional: [Ghostty](https://ghostty.org) for click-to-jump tab focus (any terminal works for the rest)

## Install (CLI / dev)

```bash
yarn install
yarn test
# one indexing pass + dashboard:
./bin/chat-manager run
# crash recovery:
./bin/chat-manager restore --dry-run
```

The placement hook (records which pane each chat is in) goes in your Claude `settings.json` under `SessionStart` and `UserPromptSubmit`:

```json
{ "type": "command", "command": "<repo>/hooks/record-placement.sh", "timeout": 5 }
```

## Build the macOS app

```bash
cd app
yarn install
yarn build      # esbuild-bundle main.ts + ../src
yarn rebuild    # rebuild better-sqlite3 for Electron's ABI
yarn dist       # -> app/release/mac-arm64/Loom.app  (unsigned)
```

Copy `Loom.app` to `/Applications` and launch it. It embeds the server, runs the summarizer + snapshots, lives in the menu bar, and launches at login.

## Notes

- The app is **unsigned** (personal use). Built locally it has no quarantine flag and opens directly.
- Privacy: your indexed chat content lives only in local files (`manager.db`, `dashboard.html`, `layout.json`, `placements.jsonl`) — all git-ignored, never committed.
